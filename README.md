# Установка и настройка VPN-сервера на протоколе Hysteria 2
Автор статьи: `VPN cat` ([я в Telegram](https://t.me/Cr0wXD), [мой Telegram канал](https://t.me/+1h4fBXuu9TE5ZDlh), [мой GitHub](https://github.com/VPN-cat))
## Введение
Эта статья - пошаговое руководство для новичков, которые хотят самостоятельно настроить свой собственный VPN-сервер на протоколе Hysteria 2. Вам не нужны глубокие знания Linux или сетевых технологий - достаточно уметь копировать команды и внимательно следовать инструкции.
#### Что такое Hysteria 2?
Это современный VPN-протокол, который специально разработан для обхода блокировок и работы в нестабильных сетях. Он очень быстрый, умеет маскироваться под обычный интернет-трафик (например, под посещение веб-сайта) и относительно легко настраивается.
> Согласно официальной документации, Hysteria 2 - прокси, а не VPN. В статье для простоты используется термин VPN, хотя технически это прокси-сервер.
#### Что нужно:
* Банковская карта для оплаты VPS.
* Немного терпения (20–30 минут).
* Доступ к ПК для настройки сервера по SSH.

>  Примечание: эта инструкция не требует навыков программирования или администрирования. Если какая-то команда выводит ошибку - перечитайте шаг, возможно, вы пропустили замену переменных (например, не изменили `<YOUR_DOMAIN>` на свой домен).

## Аренда VPS
Для настройки VPN потребуется виртуальный частный сервер (VPS). Минимальные характеристики сервера: 1 ГБ RAM, 10 GB SSD/NVMe, скорость канала от 100 Mbps и статический IPv4 адрес.

Арендуйте VPS у любого хостинг-провайдера. Локация - любая, например Европа, США или Азия (главное, чтобы не Россия или другая страна с блокировками интернета). При заказе выберите операционную систему Debian 12 (если у вашего хостинга нет варианта ОС Debian 12, то выбирайте любую более новую версию, например Debian 13). Имя пользователя и пароль для доступа к серверу по SSH обычно приходят на электронную почту или появляются в панели управления. Если вместе с именем пользователя и паролем вам отправили еще и порт (который отличается от `22`), то его нужно будет указывать при подключении.
> Вы можете найди подходящий VPS на [VDS.menu](https://vds.menu) или [Servers.expert](https://servers.Expert/)
## Подключение к серверу
Для подключения через командную строку WIndows необходима установленная Windows 10 или более новая версия. Вы также можете подключится к серверу через программу [PuTTY](https://putty.org.ru/download).
### Подключение из командной строки Windows
#### Запуск командной строки
Нажмите сочетание клавиш `Win + r`, в открывшееся окно введите `cmd` и нажмите enter. Откроется черное окно.
#### Ввод команды
В открывшееся окно введите следующую команду (замените `<SERVER_IP>` на полученный вами IP адрес вашего VPS, например `52.88.76.171`, а `<USER>` на полученое имя пользователя, например `root` или `user`):
```
ssh <USER>@<SERVER_IP>
```
Если ваш хостинг отправил вам порт для подключения по SSH (также замените `<PORT>` на ваш порт SSH):
```
ssh <USER>@<SERVER_IP> -p <PORT>
```
Примеры как может выглядеть получившиеся команда:
* `ssh root@52.88.76.171`
* `ssh user-2387@91.65.43.121 -p 2022`

>  Если команда `ssh` не найдена (например, у вас старая версия Windows), используйте PuTTY.
  После ввода команды нажмите Enter (здесь и далее после ввода каждой команды нужно нажимать Enter).
#### Добавление отпечатка ключа сервера в доверенные
При первом подключении у вас запросят добавления отпечатка ключа SSH в доверенные (IP адрес и fingerprint будут отличаться):
```text
The authenticity of host '192.168.1.90 (192.168.1.90)' can't be established.
ECDSA key fingerprint is SHA256:LNMJTbTS0EmrsGYTHB3Aa3Tisp+7fvHwZHbTA900ofw.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```
Введите `yes` и нажмите enter.
#### Ввод пароля
Далее вы должны увидеть строку вида `<USER>@<SERVE_IP>'s password:`. Введите полученный вами пароль от сервера (при вводе введенные символы не будут отображаться) и нажмите enter. 
### Подключение через PuTTY
Установите и запустите [PuTTY](https://putty.org.ru/download), в поле `Host Name (or IP address)` введите IP-адрес вашего VPS, в поле `Port` оставьте `22` или введите порт SSH который отпраил вам хостинг, в `Connection type` выберите `SSH` и нажмите `Open`. При первом подключении появится окно с предупреждением, нажмите на `Accept`. Далее у вас запроcит имя пользователя (`login as: `), введите его и нажмите Enter. В поле ввода пароля (`password`) введите ваш пароль (вводимые символы отображаться не будут) и нажмите Enter. Если все сделано верно, вы подключитесь к серверу.
## После подключения
Если подключение прошло успешно, вы увидите что-то вроде этого:
```text
Linux <HOSTNAME> 6.1.0-44-amd64 #1 SMP PREEMPT_DYNAMIC Debian 6.1.164-1 (1986-04-26) x86_64

The programs included with the Debian GNU/Linux system are free software;
the exact distribution terms for each program are described in the
individual files in /usr/share/doc/*/copyright.

Debian GNU/Linux comes with ABSOLUTELY NO WARRANTY, to the extent
permitted by applicable law.
<USER>@<HOSTNAME>:~$
```
Здесь `<HOSTNAME>` это имя вашего сервера, а `<USER>` - ваше имя пользователя. Последняя строка, которая выглядит как `<USER>@<HOSTNAME>:~$`, называется приглашением командной строки (или просто prompt). Оно приглашает вас ввести команду.
> Последний символ приглашения командной строки (`$`) может отличатся.
> Пользователь `root`  - главный администратор в Linux. У него есть абсолютная власть над системой: он может читать, изменять и удалять любые файлы, устанавливать и удалять программы, создавать других пользователей и даже сломать систему.
## Установка
### Переключение на пользователя `root`
Если ваше имя пользователя  (`<USER>`) на сервере - `root`, то пропустите этот шаг.

В дальнейшем команды на сервере следует выполнять от имени пользователя `root`. Если ваше имя пользователя на сервере не `root`, то перед выполнением команд вам нужно переключится на этого пользователя следующей командой:
```bash
sudo su - 
```
У вас запросит ваш пароль от текущего пользователя, введите его (вводимый пароль также отображаться не будет):
```text
[sudo] password for <USER>:
```
После ввода этой команды ваше приглашение изменится на `root@<HOSTNAME>:/home/<USER>#`, что означает что вы успешно переключились на пользователя `root`.

### Обновление сервера
Введите команду чтобы обновить систему и установить `curl`, `micro` и `pwgen`.
```bash
apt update && apt upgrade -y && apt install curl micro pwgen -y 
```
После ввода этой команды сервер будет обновлятся, это может занять некоторое время. Не отключайтесь от сервера во время обновения. Когда обновление завершится, вы снова увидете приглашение.
### Настройка домена
Для настройки Hysteria нам потребуется домен. Это может быть как домен любого уровня (например,  `domain.com` или `some.domain.com`). Если у вас нет собственного домена, рекомендуем использовать Dynu. 
Ниже приведены доступные варианты получения и настройки домена:
#### Собственный домен
Если у вас есть свой домен, то вы можете создать для него A-запись, указывающую
 на IP адрес вашего VPS.
#### Dynu
Зарегистрируйтесь на сайте [dynu.com](https://www.dynu.com/). Перейдите на [панель управления dynu](https://www.dynu.com/en-US/ControlPanel) и выберите "DDNS Services". На открывшейся странице нажмите "Add". Далее в разделе "Option 1: Use Our Domain Name" установите любое свободное имя хоста (Host) и домен верхнего уровня (Top Level), после чего нажмите "Add". У вас откроется страница редактирования DDNS записи. Измените значене поля "IPv4 Address" на IP адрес вашего сервера (`<SERVER_IP>`) и сохраните изменения кнопкой "Save". Запомните полученый домен.
#### FreeDNS
Вы также можете использовать [FreeDNS](https://freedns.afraid.org/).
### Установка Hysteria:
Выполните следующую команду для установки Hysteria на ваш сервер:
```bash
bash <(curl -fsSL https://get.hy2.sh/)
```
Дождитесь завершения выполнения команды. Если команда вывела `Congratulation! Hysteria 2 has been successfully installed on your server`, то все успешно установилось.

### Настройка Hysteria
#### Режимы работы Hysteria
>  Если вас не интересуют технические детали, можете пропустить этот раздел и перейти к "Настройка сайта-заглушки"

Hysteria поддерживает три основных способа маскировки трафика, чтобы обходить DPI (Deep Packet Inspection) и блокировки. В этом руководстве используется маскировка под свой сайт, но полезно понимать и другие варианты. 

##### Маскировка под свой сайт
Вы создаёте на своём VPS простой веб-сервер заглушку. Hysteria автоматически получает SSL-сертификат для вашего домена и отвечает на HTTPS-запросы этой заглушкой. Клиент Hysteria подключается к тому же домену и порту 443, но его трафик неотличим от обычного HTTPS. Для внешнего наблюдателя - просто посещение сайта.
##### Маскировка под чужой сайт
В этом режиме в настройках указывается домен известного сайта (например, `microsoft.com` или `cloudflare.com`). Если открыть ваш домен в браузере, отобразится содержимое указанного сайта. Это заставляет системы DPI думать, что вы подключаетесь к легитимному ресурсу.
##### Обфускация Salamander
Режим, при котором Hysteria не имитирует HTTPS. Вместо этого он применяет обфускацию (запутывание) каждого пакета по ключу: пакеты выглядят как случайный шум без явной структуры протокола. Salamander не создаёт веб-сервер и не использует TLS. DPI видит поток данных, который не соответствует никакому известному протоколу.

В данном руководстве выбран режим маскировки под свой сайт, так как он балансирует между надёжностью, простотой настройки и универсальностью. Если ваша страна начнёт блокировать даже такой трафик, вы можете переключиться на Salamander, изменив конфигурацию сервера и клиента (описание этого выходит за рамки этого руководства).
#### Настройка сайта для маскировки
Создайте директорию для маскировочного web-сервера:
```bash
mkdir -p /var/www/masq
```
Откройте `index.html` в редакторе `micro`:
```bash
micro /var/www/masq/index.html
```
Содержимое веб-сервера может быть любым, например мы поставим на главную страницу стандартную заглушку nginx:
```html
<!DOCTYPE html>
<html>
<head>
<title>Welcome to nginx!</title>
<style>
html { color-scheme: light dark; }
body { width: 35em; margin: 0 auto;
font-family: Tahoma, Verdana, Arial, sans-serif; }
</style>
</head>
<body>
<h1>Welcome to nginx!</h1>
<p>If you see this page, the nginx web server is successfully installed and
working. Further configuration is required.</p>

<p>For online documentation and support please refer to
<a href="http://nginx.org/">nginx.org</a>.<br/>
Commercial support is available at
<a href="http://nginx.com/">nginx.com</a>.</p>

<p><em>Thank you for using nginx.</em></p>
</body>
</html>
```
Вставьте этот текст в `micro`  используя `Ctrl+V`, далее сохраните при помощи `Ctrl+S` и закройте редактор на `Ctrl+Q`.
#### Открытие файла с конфигурацией Hysteria
Удалите стандартную конфигурацию:
```bash
rm /etc/hysteria/config.yaml
```
Откройте файл с конфигурацией Hysteria в редакторе `micro`:
```bash
micro /etc/hysteria/config.yaml
```
Измените содержимое на:
```yaml
listen: 0.0.0.0:443

acme:
  type: http
  domains:
    - <YOUR_DOMAIN>
  email: <EMAIL>

auth:
  type: userpass
  userpass:
    <HYSTERIA_USER>: <HYSTERIA_PASSWORD>

masquerade:
  type: file
  file:
    dir: /var/www/masq
  listenHTTP: :80
  listenHTTPS: :443
  forceHTTPS: true
```
Замените `<YOUR_DOMAIN>` на ваш домен, `<EMAIL>` на ваш email, который будет использован для получения сертификата, а `<HYSTERIA_USER>` и `<HYSTERIA_PASSWORD>` на логин и пароль, используемые для аутентификации при подключении к Hysteria. 

Для генерации паролей можете использовать утилиту `pwgen` :
```bash
pwgen 40
```
Здесь `40` - количество символов в пароле. Можете изменить это число, но помните, что пароль должен быть сложным

Если вы хотите чтобы доступ к VPN был у кого-то ещё, вы можете добавить пользователей (ниже приведён только раздел `auth`):
```yaml
auth:
  type: userpass
  userpass:
    my: Dahc0cu8shaikiekoo7ungieXae9aida
    friend: Aingae0Okit1eek4eeZahFohVei4akee
    yet_friend: AphiuvaetioWeite0go5iek1jai4voa9
```
Не забудьте изменит пароли из примера на свои.

Сохраните файл (`Ctrl+S`) и закройте `micro` (`Ctrl+Q`).

#### Отключение файрвола
На некоторый VPS по умолчанию включен файрвол (firewall). Чтобы VPN сервер был доступен из интернета, нужно открыть порты:
```bash
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 443/udp
```
Если выполнение этих команд выводит что-то вроде `bash: command not found: ufw` значит на сервере не включен файрвол и ничего делать не нужно.
#### Запуск Hysteria
Для запуска сервера Hysteria введите:
```bash
systemctl start hysteria-server.service
```

Чтобы посмотреть статус сервера:
```bash
systemctl status hysteria-server.service
```
Если в выводе команды есть `active (running)` то сервер успешно запустился. Чтобы выйти из просмотра статуса нажмите `Ctrl+C`.

Выполните следующую команду,[ ]()чтобы VPN-сервер запускался при загрузки VPS:
```bash
systemctl enable hysteria-server.service 
```

>  Если вы снова измените конфигурацию Hysteria, то для применения изменений нужно перезапустить:
> ```bash
> systemctl restart hysteria-server.service  
> ```

Первый запуск сервера может занять некоторое время, так как выполняется получение сертификатов. Подождите 3-5 минут, затем попробуйте открыть ваш домен (`<YOUR_DOMAIN>`) в браузере. Если сайт успешно откроется и вы не увидете никаких предупреждений о сертификатах, значит сертификаты получены успешно. Если возникнут ошибки, используйте `journalctl -u hysteria-server.service -f` для просмотра логов (`Ctrl+C` чтобы закрыть).
#### Настройка клиента Hysteria
Перейдите в домашнюю директорию:
```bash
cd
```
Создайте файл `hysteria-client.yaml`:
```bash
micro hysteria-client.yaml 
```
Вставьте в этот файл:
```yaml
server: <SERVER_DOMAIN>:443

auth: <HYSTERIA_USER>:<HYSTERIA_PASSWORD>
```
Измените `<SERVER_DOMAIN>` на домен вашего сервера, а `<HYSTERIA_USER>` и `<HYSTERIA_PASSWORD>` на логин и пароль из файла конфигурации Hysteria.
Сохраните и закройте файл.

Чтобы получить URI для вставки в VPN-клиенты выполните команду:
```bash
hysteria share -c hysteria-client.yaml
```
>  Если добавить в предыдущей команде ` --qr` то будет выведен QR код, созданый из URI.

Сохраните полученную ссылку начинающуюся с `hysteria2://`. Эту ссылку можно использовать для подключения к VPN.
## Настройка клиента Hysteria
Ниже приведены инструкции по подключению к настроенному VPN-серверу на разных операционных системах.
### Windows
Установите и запустите [Throne](https://github.com/throneproj/Throne/releases), затем нажмите "Профили" -> "Добавить профиль из буфера обмена" (предварительно скопируйте полученную ссылку). Кликните по появившемуся профилю и нажмите Enter, после чего Throne подключится к вашему VPN-серверу.
### macOS
Используйте [Throne](https://github.com/throneproj/Throne/releases/tag/1.1.1), о том как его наcтроить см. пункт Windows.
### Linux 
На Linux вы также можете использовать [Throne](https://github.com/throneproj/Throne/releases/tag/1.1.1), о том как его наcтроить см. пункт Windows.
### Android 
Установите приложение [v2rayNG](https://github.com/2dust/v2rayng) на ваше устройство. Скопируйте вашу ссылку в буфер обмена, откройте v2rayNG, нажмите "+" в правом верхнем углу, выберите "Импорт из буфера обмена". У вас появится новый VPN-профиль, выберите его. Затем подключитесь к VPN, нажав на кнопку подключения внизу приложения. Дождитесь появления надписи "Службы успешно запущены", которая означает успешное подключение.
### iOS
Установите [Hiddify](https://apps.apple.com/us/app/hiddify-proxy-vpn/id6596777532). Откройте приложение, нажмите "Добавить новый профиль". Выберите "Импорт из буфера обмена" (предварительно скопируйте ссылку для подключения). Нажмите на появившийся профиль и выберите "Подключиться". При первом подключении система запросит разрешение на добавление VPN-конфигурации - нажмите "Разрешить".
